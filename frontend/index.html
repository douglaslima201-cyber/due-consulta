Como funciona

Quatro etapas automáticas, do upload ao relatório final.

01 —
📊

Upload da planilha

Envie seu arquivo .xlsx ou .csv com chaves de 44 dígitos. Detecção automática de coluna e remoção de duplicatas.

02 —
✅

Validação

Cada chave é validada estruturalmente antes do envio. Chaves inválidas são listadas em log separado com preview.

03 —
🤖

Automação Siscomex

Playwright consulta o portal para cada chave, resolve CAPTCHAs automaticamente e extrai os dados da DUE vinculada.

04 —
📥

Relatório Excel

Resultados consolidados em planilha com código de cores por status, aba de resumo e histórico completo.

🔄

Retry automático

3 tentativas por chave com backoff. Falhas por timeout, CAPTCHA ou instabilidade do portal são tratadas automaticamente.

💾

Checkpoint contínuo

Cada resultado é salvo imediatamente em SQLite local. Interrupções não perdem dados já processados.

📋

Log auditável

Cada etapa do processamento é registrada com timestamp, nível de severidade e descrição completa.

🔐

100% local

Nenhum dado é enviado a servidores externos. Tudo roda no seu computador — uploads, consultas e resultados.

⚡

Anti-Captcha integrado

Resolução automática via API anti-captcha.com. Fallback para resolução manual com aviso visual claro.

📊

Relatório profissional

Excel formatado com código de cores por status, aba de resumo com métricas e colunas otimizadas.

Guia de uso

Documentação

Instalação, configuração e operação completas.

⚙️ Instalação rápida

# 1. Instalar dependências cd backend pip install -r requirements.txt # 2. Instalar navegador python -m playwright install chromium # 3. Iniciar servidor python main.py # 4. Abrir frontend/index.html

📊 Estrutura da planilha

  • Coluna com "chave", "nfe" ou "nota" no nome
  • Ou qualquer coluna com strings de 44 dígitos
  • Formatos aceitos: .xlsx e .csv
  • Duplicatas removidas automaticamente
  • Chaves inválidas listadas em log separado

📥 Campos do relatório

  • Chave NF-e (44 dígitos)
  • Status: Averbada / Não Averbada / Não Encontrada
  • Número da DUE vinculada
  • Data e status da DUE
  • Observações e timestamp da consulta

🔌 Endpoints da API

  • POST /api/upload — Validar planilha
  • POST /api/iniciar/{id} — Iniciar job
  • GET /api/status/{id} — Status em tempo real
  • POST /api/cancelar/{id} — Cancelar job
  • GET /api/download/{id} — Baixar Excel
  • GET /api/jobs — Listar histórico
Perguntas frequentes

FAQ

O backend precisa estar rodando para usar a ferramenta?+
Sim. O frontend se comunica com o servidor Python local na porta 5000. Execute python main.py na pasta backend/ antes de abrir a interface.
O que acontece se o CAPTCHA não for resolvido automaticamente?+
O Chromium será exibido na tela para resolução manual. Após resolver, a consulta continua automaticamente para as demais chaves sem intervenção adicional.
Meus dados ficam em algum servidor externo?+
Não. Toda execução é local. Os únicos dados enviados externamente são os CAPTCHAs para resolução via anti-captcha.com — nenhuma chave de NF-e ou resultado de consulta sai do seu computador.
Posso retomar após uma interrupção?+
Os resultados processados ficam no banco SQLite local. Em caso de interrupção, baixe o relatório parcial com os dados já coletados via botão "Baixar relatório".
O portal mudou o layout e a extração parou. O que fazer?+
Screenshots de debug são salvos automaticamente em backend/. Os seletores CSS/XPath podem ser ajustados na função consultar_chave() no arquivo main.py.
